# Работа с БСП (Библиотека стандартных подсистем / Standard Subsystems Library, SSL)

> Этот файл применим **только** если ваш проект использует БСП. Если БСП не используется — удалите ссылку на этот файл из `INDEX.md`.

## Главный принцип

**Перед написанием собственного кода — проверь, нет ли решения в БСП.** Своя реализация поверх готового модуля БСП — это технический долг.

## Workflow для новой функциональности

1. Поиск в БСП — `search_in_code` по проекту с фильтром по общим модулям, или `get_metadata_objects` с `metadataType: commonModules` и `nameFilter` по тематическому слову (`Files`, `Email`, `Print`, `Users`, …).
2. Изучи методы найденного модуля — `get_module_structure`, `read_method_source` для интересных.
3. Если решение есть — используй его.
4. Только если решения нет — пиши своё. И в комментарии объясни, почему БСП не подошла.

| Модуль | Назначение |
|---|---|
| `Пользователи` | Пользователи, роли, права доступа |
| `РаботаСФайлами` | Файлы и присоединённые объекты |
| `УправлениеПечатью` | Печатные формы |

| `РаботаСПочтовымиСообщениями` | Отправка почты |

| `СтроковыеФункцииКлиентСервер` | Работа со строками |
| `РаботаСКурсамиВалют` | Курсы валют |

## Часто используемые утилиты

| Функция | Назначение |
|---|---|
| `ОбщегоНазначенияКлиентСервер.СвойствоСтруктуры(Стр, "Ключ", Умолч)` | Безопасный доступ к полю структуры |
| `ОбщегоНазначения.МенеджерОбъектаПоСсылке(Ссылка)` | Менеджер по ссылке (вместо `Выполнить("Справочники." + Имя)`) |
| `ОбщегоНазначения.ПодсистемаСуществует(Имя)` | Проверка существования подсистемы перед условным вызовом модуля |
| `ОбщегоНазначения.ОбщийМодуль(Имя)` | Получение модуля по имени для динамического вызова |
| `ОбщегоНазначения.ЭтоСсылка(Значение)` | Проверка типа — ссылка ли |
| `ОбщегоНазначения.СсылкаСуществует(Ссылка)` | Существует ли объект в БД |
| `ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Ссылка, "Реквизит")` | Получение реквизита по ссылке без выгрузки всего объекта |
| `СтроковыеФункцииКлиентСервер.СтрокаСЧисломПредметов(Кол, "документ, документа, документов")` | Склонение слова по числу |

## Конвенция суффиксов модулей

| Суффикс | Среда | Кеш | Пример |
|---|---|---|---|
| (без суффикса) | Сервер | Нет | `ОбщегоНазначения` |
| `Клиент` | Клиент | Нет | `ОбщегоНазначенияКлиент` |
| `КлиентСервер` | Обе | Нет | `ОбщегоНазначенияКлиентСервер` |
| `ПовтИсп` | Сервер | На сессию | `ОбщегоНазначенияПовтИсп` |

- В клиентском коде сначала ищи `*КлиентСервер`, потом `*Клиент`.
- В серверном — базовый модуль (без суффикса) или `*Сервер`.
- `*ПовтИсп` — кеш на сессию, для часто запрашиваемых справочных данных.

## Маркеры модификации БСП

Любое изменение модуля БСП **должно** быть обёрнуто в маркеры. Это нужно, чтобы при обновлении БСП было видно, какие места переписывались.

### Формат маркеров

- Открытие: `// <<МОДИФИКАЦИЯ` (без пробела между `<<` и словом). Опционально — короткая заметка через пробел или ` - `: `// <<МОДИФИКАЦИЯ - Расширен список адресатов`.
- Закрытие: `// >>`.
- Для однострочной правки (изменённая сигнатура и т. п.) допустим только trailing-маркер: `Процедура ИмяПроцедуры(...) Экспорт // <<МОДИФИКАЦИЯ` — без `// >>`.
- Маркеры — это **merge-метаданные**, не журнал. Не пиши в них номера задач, даты, авторов — для этого есть git.

> **Замечание:** В разных компаниях/командах принято свой префикс (например, `<<ACME`, `<<КОМПАНИЯ`). Согласуй префикс внутри команды и используй его единообразно.

### Сохранение оригинала

При **замене** или **удалении** строки БСП — оставляй оригинал закомментированным внутри блока. При обновлении БСП это покажет, что именно было заменено и на что.

```bsl
// <<МОДИФИКАЦИЯ
//	ШаблоныЗаданий.Добавить(Метаданные.РегламентныеЗадания.ПолучениеИОтправкаЭлектронныхПисем.Имя);
ШаблоныЗаданий.Добавить("ПолучениеИОтправкаЭлектронныхПисем");
// >>
```

Для чистых **добавлений** (никакая строка БСП не заменяется) — просто оборачиваем новый код:

```bsl
// <<МОДИФИКАЦИЯ - Заполнение списка доступных для связи документов
СписокДокументов.Добавить("#Задание", НСтр("ru = 'Задание'"));
СписокДокументов.Добавить("#ЗаказКлиента", НСтр("ru = 'Заказ клиента'"));
// >>
```

При чистых **удалениях** — удалённую строку БСП оставляем закомментированной внутри маркеров, без замены.

### Когда применять

- Любая правка `.bsl` модуля БСП (`СтандартныеПодсистемы.*`, `ОбщегоНазначения`, `Пользователи`, `РаботаСФайлами` и т. д.).
- Правки в библиотеках-расширениях, которые вы обновляете извне (например, библиотеки телефонии) — та же конвенция.
- Свои модули проекта — маркеры **не нужны**, это ваш код, при обновлении БСП он не пострадает.

## Префикс `Удалить` — устаревшие объекты

В конфигурациях на базе БСП объекты метаданных и реквизиты с префиксом `Удалить` (например, `УдалитьКонтрагент`, `УдалитьАдрес`) — это deprecated-заглушки, оставленные для обратной совместимости. Считай их несуществующими: не читай, не пиши, не ссылайся и не расширяй их в новом коде.
