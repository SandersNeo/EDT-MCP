# Стиль и качество BSL-кода

> Адаптировано из проверенных правил рабочих 1С-проектов. Не привязано к конкретной конфигурации.

## Комментарии

- Предпочитай **самодокументирующийся код** комментариям. Не комментируй то, что код и так выражает.
- Комментарий уместен, когда добавляет: мотивацию, описание нетривиального алгоритма, ограничения/побочные эффекты, маркер технического долга.
- Не пиши в BSL-комментариях номера трекер-задач — их место в коммите/PR, в коде они быстро устаревают.

## Качество и проверки

- После любой правки выполняй внутренний код-ревью (стиль, читаемость, корректность, edge-cases, безопасность, конкурентность). Если нашёл проблему — исправь и повтори, пока код не чист.
- Перед написанием нового кода — проверь, нет ли в общих/менеджерских модулях уже экспортной реализации.
- При написании запроса — сверь имена реквизитов и табличных частей с метаданными.

## Запреты

- Не использовать `Сообщить()`. Использовать `ОбщегоНазначения.СообщитьПользователю` (сервер) или `ОбщегоНазначенияКлиент.СообщитьПользователю` (клиент).
- Не использовать deprecated глобальные методы (`ПодробноеПредставлениеОшибки()` без префикса). Использовать `ОбработкаОшибок.ПодробноеПредставлениеОшибки()`.
- Не использовать букву `ё`/`Ё` (U+0451/U+0401) в `.bsl` файлах. Использовать `е`/`Е`. В пользовательской документации — допустимо.
- Не использовать длинное тире `—` (U+2014) в `.bsl`. Использовать `-` (U+002D).
- Не обращаться к реквизитам ссылок через точку (`Контрагент.ИНН`) — это вытягивает весь объект. Использовать `ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Контрагент, "ИНН")`.
- Не делать запросы внутри циклов. Использовать пакетные запросы и временные таблицы.
- Не сравнивать булево значение с `Истина`/`Ложь`. Писать `Если Активен Тогда`, а не `Если Активен = Истина Тогда`.
- Не использовать имена глобального контекста как имена переменных (`Документы`, `Справочники`, `Метаданные`, `Константы`).
- Не называть процедуры/функции/переменные именами встроенных функций (`Выполнить`, `Тип`, `Строка`, `Дата`, `Формат`, `ОписаниеОшибки`).
- Не использовать `Попытка ... Исключение` для операций чтения/записи БД без обоснования.
- Не добавлять `ЗаписьЖурналаРегистрации()` в обычный код без явной просьбы. **Исключение** — внутри `Исключение` блока транзакции (см. секцию «Транзакции», там запись в журнал — часть канонического паттерна).
- Не использовать венгерскую нотацию (`МассивКонтрагентов`). Писать `Контрагенты`.
- Не сокращать имена объектов и реквизитов до аббревиатур (`АВР`). Полные имена: `АктВыполненныхРабот`.

## Запросы 1С

- Текст запроса — на новой строке после `Запрос.Текст =`, на том же уровне отступа:
  ```bsl
  Запрос = Новый Запрос;
  Запрос.Текст =
      "ВЫБРАТЬ
      |   Контрагенты.ИНН КАК ИНН
      |ИЗ
      |   Справочник.Контрагенты КАК Контрагенты";
  ```
- Всегда используй параметры (`Запрос.УстановитьПараметр()`), не подставляй значения конкатенацией строк.
- Всегда алиасы через `КАК`: `Контрагенты.ИНН КАК ИНН`.
- Используй `ПЕРВЫЕ N`, если нужно ограниченное число записей.
- Промежуточная переменная для результата: `РезультатЗапроса = Запрос.Выполнить()`. Не цепляй `.Выполнить().Выгрузить()`.
- Перед вставкой нового запроса в код — `validate_query`.

## Транзакции — канонический паттерн

```bsl
НачатьТранзакцию();
Попытка
    // Блокировка (если нужна) - Чтение - Запись
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ЗаписьЖурналаРегистрации(НСтр("ru = 'Событие'"),
        УровеньЖурналаРегистрации.Ошибка, , ,
        ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
    ВызватьИсключение;
КонецПопытки;
```

Правила:
- `НачатьТранзакцию()` непосредственно перед `Попытка`. Между ними — никакого кода.
- `ЗафиксироватьТранзакцию()` — последний оператор перед `Исключение`.
- `ОтменитьТранзакцию()` — первый оператор в `Исключение`.
- `ЗаписьЖурналаРегистрации` — **после** `ОтменитьТранзакцию()` (иначе пропадёт при откате).
- Всегда `ВызватьИсключение` после логирования — пробрасываем выше.
- Внутри транзакции — никаких HTTP-вызовов, тяжёлых запросов, диалогов с пользователем.

## Блокировки

- `БлокировкаДанных` — перед чтением данных в транзакции, чтобы избежать race condition (read-lock-modify-write).
- `ЗаблокироватьДанныеДляРедактирования(Ссылка)` — при программном изменении объекта (защита от lost update).
- Порядок блокировок — единый по проекту (защита от взаимоблокировок).

## Перевыброс исключений

- `ВызватьИсключение;` без параметра — сохраняет оригинальный stack trace. Использовать в промежуточном коде.
- `ВызватьИсключение "Текст";` — новое исключение, стек теряется. Использовать на границе с пользователем.
- `КраткоеПредставлениеОшибки()` — для сообщений пользователю, `ПодробноеПредставлениеОшибки()` — для лога.

## Производительность строк

В циклах — `Массив` + `СтрСоединить()` вместо `+`:

```bsl
ЧастиСтроки = Новый Массив;
Для Каждого Элемент Из Коллекция Цикл
    ЧастиСтроки.Добавить(Элемент.Наименование);
КонецЦикла;
Результат = СтрСоединить(ЧастиСтроки, ", ");
```

Для больших HTML/CSS-шаблонов — `ПотокВПамяти` + `ЗаписьТекста.ЗаписатьСтроку()`.

## Локализация

- Не строй пользовательские строки конкатенацией. Используй `СтрШаблон` + `НСтр`, вынеси результат в переменную:
  ```bsl
  // НЕПРАВИЛЬНО
  Результат = НовыйРезультат(Истина, "Записано для " + Количество + " пользователей");

  // ПРАВИЛЬНО
  ТекстРезультата = СтрШаблон(НСтр("ru = 'Записано для %1 пользователей'"), Количество);
  Результат = НовыйРезультат(Истина, ТекстРезультата);
  ```

## Даты

- `ТекущаяДатаСеанса()` вместо `ТекущаяДата()` — корректная работа с часовыми поясами.

## Формы

- Предпочитай `&НаСервереБезКонтекста` — не сериализует весь контекст формы (для больших таблиц это мегабайты).
- Группируй серверные вызовы: один `&НаСервереБезКонтекста`, возвращающий `Структура`, вместо нескольких раздельных.
- После `РеквизитФормыВЗначение("Объект")` **обязательно** `ЗначениеВРеквизитФормы(ОбъектДокумент, "Объект")`, иначе изменения теряются.
- Изменения видимости/доступности — в одной серверной процедуре `УправлениеВидимостью()`.
- Использовать асинхронные диалоги (`ПоказатьВопрос` + `ОписаниеОповещения`) вместо модальных (`Вопрос()`) — модальные не работают в веб-клиенте.
- `Оповестить()` / `ОбработкаОповещения()` — для коммуникации между формами после записи.

## Назначение регионов модуля

- `ПрограммныйИнтерфейс` — публичные методы.
- `СлужебныйПрограммныйИнтерфейс` — приватные методы для внутренних обращений.
- `СлужебныеПроцедурыИФункции` — внутренняя реализация.

## Размер процедур и модулей

- Процедуры/функции — до 200 строк. Свыше 100 — подумай о декомпозиции.
- При добавлении нового кода в общий модуль — если в нём уже ~3000 строк, создай новый сфокусированный модуль.
- Существующие большие модули не рефактори без явной просьбы пользователя.

## Документирование

- Документируй публичные процедуры/функции: назначение, параметры, возвращаемое значение.

## Особенности языка BSL

- `Неопределено` vs `Null`. `Неопределено` — отсутствие значения переменной (default для необъявленного параметра, неинициализированной переменной, отсутствующего ключа структуры в `СвойствоСтруктуры`). `Null` — отдельный тип, который приходит **только** из СУБД для отсутствующего значения поля (LEFT JOIN, ЕСТЬNULL и т. п.). Не путай: `Структура.Свойство("X") = Неопределено`, а `Выборка.Сумма = Null` для пустого JOIN. Проверять надо тем же, чем пришло: `Если Значение = Неопределено Тогда` или `Если Значение = Null Тогда` (либо `ЗначениеЗаполнено(Значение)`).
- `ЗначениеЗаполнено(Значение)` — единая проверка «значение заполнено» (не пустая строка, не пустая ссылка, не `Неопределено`, не `Null`, не нулевая дата). Предпочитай её ручному `Если Значение <> "" И Значение <> Неопределено`.
- `Тип("СправочникСсылка.Контрагенты")` vs `ТипЗнч(Значение)`. Первый — получить объект-тип по имени. Второй — узнать тип значения. Сравнение: `ТипЗнч(Ссылка) = Тип("СправочникСсылка.Контрагенты")`.
- `Новый Структура("А, Б", ЗначА, ЗначБ)` — короткая инициализация. Имена ключей — без пробелов, через запятую одной строкой.
- Все строковые литералы для пользователя — через `НСтр("ru = '...'; en = '...'")`. Никогда не складывай куски пользовательского сообщения через `+`, см. секцию «Локализация».

## Привилегированный режим

- `УстановитьПривилегированныйРежим(Истина)` — только когда нужно. После — обязательно `УстановитьПривилегированныйРежим(Ложь)`.
